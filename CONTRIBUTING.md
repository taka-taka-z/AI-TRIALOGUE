# コントリビューションガイド

AI TRIALOGUEへの貢献に感謝します。

## 貢献の方法

### バグ報告

1. [Issues](../../issues) を開いてください
2. 以下の情報を含めてください：ブラウザ名とバージョン、再現手順、期待される動作と実際の動作
3. API関連の場合はエラーメッセージを含めてください（APIキーは必ず伏せてください）

### 機能提案

`enhancement` ラベルを付けてIssueを作成してください。ユースケースと期待される動作を記載してください。

### プルリクエスト

1. リポジトリをフォーク
2. フィーチャーブランチを作成（`git checkout -b feature/your-feature`）
3. `index.html` を編集（単一ファイルアーキテクチャ）
4. Chrome・Firefox・Safariでテスト
5. 変更内容の説明を添えてPRを提出

## 設計原則

- **単一ファイル** — すべてを1つのHTMLファイルに。ビルドステップなし
- **依存ゼロ** — npm不要、バンドラ不要。ブラウザだけで動作
- **プライバシー最優先** — APIキーはブラウザの外に出さない
- **アクセシビリティ** — WCAG AAコントラスト比、キーボードナビゲーション、aria属性
- **クリーンUI** — 絵文字アイコン不使用、紫グラデーション不使用、カード入れ子構造不使用

## コードスタイル

- 幅広いブラウザ互換性のため `var` / `function` を使用
- すべての色・余白にCSS変数（カスタムプロパティ）を使用
- 適切な `aria-*` 属性を持つセマンティックHTML
- UI表示文字列は日本語、コードロジックのコメントは日英混在可
